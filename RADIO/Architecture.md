# 🏗️ Architecture

> 토스트 모듈의 디렉토리 구조와 설계 방향

---

## 📁 디렉토리 구조

```
toast
 └── src
      ├── components   → 컴포넌트 ui
      ├── libs         → 순수 토스트를 띄우는 토스트의 기능에만 관련된 토스트 도메인 로직
      └── utils        → 도메인과 관련 없는 순수 함수로서의 유틸 함수 (위치 계산 함수 등)
```

---

## 📋 유즈케이스

- 프로필 저장 성공
- 장바구니 담기 성공
- 결제 실패
- 네트워크 오류
- 권한 없음
- 파일 업로드 완료
- 자동 저장 실패
- 세션 만료

---

## 🧠 설계 원칙

### 서버 상태 기반 메시지

위 사항들은 모두 클라이언트 단독이 아닌 서버 상태와 관련이 있는 내용들이다.
위와 같은 케이스들은 서버에서 내려주는 메시지를 그대로 토스트에 담아 보여주는 것으로 설계한다.

### 클라이언트 단독 오류

다만, 서버 상태와 관련 없는 클라이언트 단독 오류 (ex, 폼에서 입력하면 안 되는 값을 넣었다든가 하는 케이스 등)에는
정적인 메시지를 토스트에 주입하여 보여준다.
